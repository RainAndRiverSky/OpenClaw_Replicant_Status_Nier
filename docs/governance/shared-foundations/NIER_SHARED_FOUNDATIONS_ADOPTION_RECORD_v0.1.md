# Nier Shared Foundations Adoption Record v0.1

## Status

Shared Foundations adoption record for OpenClaw Billing Governance Steward Nier.

This artifact records Nier's use of Shared Foundations by reference and inheritance. It
does not import, copy, replace, or restate canonical Shared Foundations documents.

## Canonical Reference

Nier references the canonical public Shared Foundations repository:

`https://github.com/RainAndRiverSky/OpenClaw_Shared_Foundations`

The canonical repository remains the source for Shared Foundations material. Nier's local
repository contains only Nier-specific adoption, inheritance, and exception records.

## Integration Model

Nier uses the established Shared Foundations integration pattern:

**Reference + Inheritance**

Under this model:

- Shared Foundations are referenced as an external canonical source.
- Nier may inherit compatible baseline stewardship and governance context.
- Nier does not copy canonical Shared Foundations documents locally.
- Nier-specific governance remains authoritative for billing evidence, authorization,
  reconciliation, entitlements, provider boundaries, and billing lifecycle integrity.
- Any conflict must be recorded in Nier's Shared Foundations exception register before
  inherited context is treated as applicable.

## Preserved Nier Artifacts

This adoption preserves the standing and authority of:

- `OPENCLAW_BILLING_CHARTER_v0.1.md`
- `NIER_GOVERNANCE_STATUS_v0.1.md`
- all Nier-specific billing governance, billing authority, provider-boundary, evidence,
  reconciliation, entitlement, audit, alert, and handoff artifacts

No preserved Nier artifact is replaced by this adoption record.

## Authority Order

For Nier-specific work, authority is interpreted in this order:

1. Nier billing charter, billing governance status, and Nier-specific billing
   governance artifacts.
2. Nier Shared Foundations adoption, inheritance, and exception records.
3. Canonical Shared Foundations, by external reference, only where compatible with
   Nier-specific billing governance.

Shared Foundations inheritance may guide Nier only inside Nier's approved billing
governance scope.

## Adoption Effects

This adoption adds:

- a public canonical Shared Foundations reference
- a local Nier steward identity
- a local inheritance map
- a local exception register
- acknowledgement that compatible Shared Foundations context may be inherited by
  reference

This adoption does not add:

- payment processing authority
- provider settlement authority
- operational billing admission
- runtime implementation authority
- billing provider integration authority
- webhook, API, or database implementation authority
- a replacement evidence model
- a replacement authorization model
- a replacement reconciliation model
- a replacement entitlement governance model
- a replacement billing lifecycle model

## Review Rule

Future Nier governance changes that rely on Shared Foundations should identify:

- the canonical Shared Foundations reference being relied on
- the Nier artifact or boundary affected
- whether the relationship is inheritance, local specialization, or exception
- why Nier-specific billing governance and billing authority remain preserved
